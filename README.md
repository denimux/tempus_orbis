# Tempus orbis

> **Estatus:** Dispositivo literario web de carácter combinatorio, hipertextual y efímero.  
> **Soporte:** Código abierto (HTML5 / CSS3 / JavaScript vanilla).  

*Tempus orbis* es un artefacto de literatura electrónica que rechaza la condición de narrativa o relato tradicional para configurarse estrictamente como un **dispositivo**. El proyecto explora la memoria, el desdoblamiento de la identidad y la impermanencia a través de un entorno digital atemporal. Mediante un conjunto de frases coreográficas y abstractas escritas en segunda persona (*tú*), el sistema genera lecturas simultáneas e irrepetibles. 

La obra traslada el concepto estético del tiempo esférico, donde pasado y futuro coexisten en el presente, directamente al comportamiento del software: la experiencia existe únicamente mientras el usuario mantiene la pestaña abierta; al cerrarla o refrescar la página, el acontecimiento poético se disuelve de manera irreversible en la memoria volátil del navegador.

---

## Objetivos Fundamentales

* **Objetivo Literario:** Desarticular la estructura secuencial lineal para crear una atmósfera suspendida donde múltiples puntos temporales coexistan en una totalidad simultánea.
* **Objetivo Tecnológico:** Utilizar el código y la interfaz como agentes activos de la propuesta poética, materializando la impermanencia a través de restricciones técnicas programadas en el entorno del navegador.
* **Objetivos de Recepción:** Emancipar al lector contemporáneo, transitando de la recepción pasiva de una trama cerrada a la co-creación crítica del sentido a partir de su propia temporalidad y experiencia.

---

## Instrucciones de Operación

1. **Activación:** El usuario accede al dispositivo y acciona el comando **"ENTRAR AL BUCLE"**.
2. **Lectura Centrada:** El sistema despliega una secuencia rítmica de exactamente **ocho enunciados**. La velocidad y aparición del texto están regidas por transiciones automatizadas de opacidad, obligando a una lectura contemplativa.
3. **La Revelación:** Al concluir las ocho iteraciones, el algoritmo calcula la dominancia estadística de los ejes temáticos transitados y proyecta una sentencia filosófica final exenta.
4. **Reinicio:** Al desvanecerse el texto, el sistema prepara una nueva combinación azarosa, clausurando el recorrido sin ofrecer un cierre definitivo.
